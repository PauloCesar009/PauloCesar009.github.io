AJAX (Asynchronous Javascript And XML)

é uma técnica de desenvolvimento web usada para criar aplicações interativas e dinâmicas. Com AJAX, é possível atualizar partes de uma página da web sem recarregar a página inteira. Isso melhora a experiência do usuário, tornando as interações mais rápidas e fluidas.

Embora o nome inclua "XML", hoje em dia é mais comum usar JSON (JavaScript Object Notation) em vez de XML, por ser mais leve e fácil de trabalhar. A técnica AJAX envolve o uso de várias tecnologias em conjunto

Promises

Promises são um recurso do JavaScript usado para gerenciar operações assíncronas de forma mais elegante do que com callbacks tradicionais. Eles representam um valor que pode estar disponível agora, no futuro, ou nunca, e fornecem uma maneira estruturada de lidar com o sucesso ou falha de uma operação assíncrona.

Fetch API

A Fetch API é uma interface moderna do JavaScript que facilita a realização de requisições HTTP, como buscar ou enviar dados de/para um servidor. Ela é preferida em relação ao XMLHttpRequestpor ser mais simples, baseada em Promessas , e oferecer uma maneira mais limpa e concisa para lidar com operações assíncronas.

Async / Await

Async/Await é uma maneira mais simples e legível de lidar com operações assíncronas no JavaScript, introduzida no ECMAScript 2017. Ela funciona em cima de Promises , mas permite que você escreva o código assíncrono de maneira que manifesta síncrona, evitando o uso excessivo de .then()e .catch().

async : Define uma função como assíncrona. Toda função marcada com asyncretorna uma Promise automaticamente, mesmo que você não retorne uma Promise explicitamente.

await : O operador awaitsó pode ser usado dentro das funções async. Ele pausou a execução da função até que a Promise seja resolvida ou rejeitada. Quando a Promessa é resolvida, o valor é devolvido; se rejeitada, um erro é lançado.
